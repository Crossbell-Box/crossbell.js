Class Contract

This class is used to interact with the contract.

example

Connect with Metamask

import { Contract } from 'crossbell.js'

// Create a new contract instance with metamask provider
const provider = window.ethereum
const contract = new Contract(provider)

// Connect to the chain
await contract.connect()

// Example API: Create a new profile for an address
try {
  const result = await contract.createProfile(
    '0x1234567890123456789012345678901234567890',
    'Jason',
    'ipfs://xxxx/metadata.json',
  )
  console.log(result.data) // '42' (profileId)
  console.log(result.transactionHash) // '0xabcdef...'
} catch (e) {
  console.error(e.message) // e.g. "execution reverted: Web3Entry: HandleExists"
}

Link Methods

createThenLinkProfile

  • createThenLinkProfile(fromProfileId: string, toAddress: string, linkType: string): Promise<Result<{ linklistId: string; toProfileId: string }, true>>

  • Inherited from Contracts.createThenLinkProfile

    • Defined in contract/subcontracts/link.ts:115

    This creates a profile for an target address and links the fromProfile to it.

    This should be only called when the target address doesn't have any profile. When called on an address that already has a profile, this will fail. When called, this will create a new profile for the target address and set the new profile as the primary profile for this address. The new profile's handle will be set to the address of the target address.

    Parameters

    • fromProfileId: string

      The profile ID of the profile that is creating the new profile. Must be your own profile, otherwise it will be rejected.

    • toAddress: string

      The address of the profile you want to link to.

    • linkType: string

      The type of link you want to create. This is a string.

    Returns Promise<Result<{ linklistId: string; toProfileId: string }, true>>

    The transaction hash of the transaction that was sent to the blockchain, the toProfileId and linklistId.

changeProfileMetadata

  • changeProfileMetadata(profileId: string, modifier: (metadata?: ProfileMetadata) => ProfileMetadata): Promise<Result<{ metadata: ProfileMetadata; uri: string }, true>>

  • Inherited from Contracts.changeProfileMetadata

    • Defined in contract/subcontracts/profile.ts:154

    This changes a profile's metadata (URI).

    example

    change a profile's metadata name and bio

    await contract.changeProfileMetadata('42', metadata => {
      if (metadata !== undefined) {
        metadata.name = 'John Doe'
        metadata.bio = 'I am a person'
      } else {
        metadata = {
          name: 'John Doe',
          bio: 'I am a person',
        }
      }
      return metadata
    })
    example

    change a profile's metadata name and bio (using spread operator)

    await contract.changeProfileMetadata('42', metadata => {
      metadata = {
        ...metadata,
        name: 'John Doe',
        bio: 'I am a person',
      }
      return metadata
    })

    Parameters

    • profileId: string

      The profile ID of the user you want to set the URI for.

    • modifier: (metadata?: ProfileMetadata) => ProfileMetadata

      The callback function that modifies the metadata.

          Returns ProfileMetadata

    Returns Promise<Result<{ metadata: ProfileMetadata; uri: string }, true>>

Revision Methods

checkRevision

  • checkRevision(): Promise<{ data: { currentRevision: number; isUpToDate: boolean; latestRevision: number } }>

  • Inherited from Contracts.checkRevision

    • Defined in contract/subcontracts/revision.ts:43

    This checks if the SDK is up-to-date with the contract. Although the contract will update its revision from time to time, it will keep compatibility with the old revision. I.e. Crossbell's developers ensured that there is not going to be any breaking changes when introducing new features in the future.

    Therefore, this method is used to check if you need to update your SDK to use some new features.

    Returns Promise<{ data: { currentRevision: number; isUpToDate: boolean; latestRevision: number } }>

    Whether the SDK is up-to-date with the contract and the local/remote revision.
